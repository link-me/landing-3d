# 3D Landing Page (Three.js + WebGL)

Интерактивный 3D-лендинг на базе Three.js, демонстрирующий:
- рендеринг WebGL-сцены с анимациями,
- орбитальные контролы для обзора модели,
- параллакс-эффект при прокрутке,
- переключение темы (тёмная/светлая),
- простые метрики производительности (FPS, количество объектов),
- реалистичные столкновения между парой «капсул» (пилюль), которые строго движутся по своим орбитам вокруг ядра.

Stack: Three.js + WebGL (+ Vite для локальной разработки)

## Возможности
- Две 3D-капсулы («пилюли»), вращающиеся вокруг ядра по наклонённым орбитам.
- Столкновения пилюль с импульсной моделью: нормальная и касательная компоненты, спин и отскок.
- Гарантированная коррекция проникновения с итерациями: объекты не проходят друг через друга.
- Строгая орбита: радиальная компонента смещения к ядру запрещена, возврат на траекторию с демпфированием.
- Светящийся акцент при столкновениях (PointLight на пилюле) и лёгкий «вертикальный» визуальный kick.
- Орбитальные контролы (OrbitControls) с демпфированием; параллакс по скроллу.
- Переключатель темы, базовые метрики FPS/объектов, адаптивный ресайз.

## Установка и запуск
Требуется Node.js >= 18.

```bash
npm install
npm run dev
```

Откройте выданный локальный URL в браузере. Для production-сборки:

```bash
npm run build
npm run preview
```

## Структура
- `index.html` — оболочка приложения, элементы UI.
- `src/main.js` — инициализация Three.js сцены, анимации и интерактив.
- `README.md` — описание проекта и инструкции.
- `ROADMAP.md`, `ROADMAP.en.md` — план развития на двух языках.
- `.gitignore` — служебные исключения, включая локальные файлы.

## Что решает
- Демонстрирует аккуратную физику столкновений без тяжёлого физдвижка и при этом сохраняет управляемые орбиты.
- Даёт основу для наращивания эффектов (шейдеры, постобработка, дополнительные объекты и взаимодействия).
- Обеспечивает минимальный UX (темы, CTA, метрики) и простую расширяемость.

## Параметры физики (в `src/main.js`)
- `RESTITUTION` — упругость отскока (0..1), по умолчанию ≈ 0.85.
- `FRICTION` — касательная «сцепление» для передачи спина, по умолчанию ≈ 0.25.
- `OFFSET_DAMP` — демпфирование локальных скоростей смещения, по умолчанию ≈ 0.94.
- `VEL_MAX` — ограничение свободной локальной скорости, чтобы избежать туннелирования.
- Итерации коррекции проникновения — до 4 за кадр для надёжного разведения.

## Ограничения
- Столкновения с ядром не моделируются: пилюли строго удерживаются на орбитах (без входа в ядро).
- Импульсы ограничиваются cooldown-ом, но позиционная коррекция — всегда: пересечение не остаётся.
- Радиальное смещение к ядру отключено, смещения — только по касательной (локальные Y/Z).

## Технические замечания
- Используется `OrbitControls` из `three/examples/jsm`.
- Цветовое пространство рендера — `SRGBColorSpace`.
- Анимации рассчитаны по времени кадра; рендер — на `requestAnimationFrame`.
- Параллакс основан на текущем `scrollY` относительно высоты вьюпорта.
